LLM Speedrun Submission Summary

Project Overview

The objective of this project was to improve the baseline language model while respecting the competition constraints of fewer than 2 million parameters, exactly 2000 optimizer steps, CPU-only training and inference, and training only on the provided dataset.

The work primarily focused on identifying modifications that improve the Bits-Per-Byte (BPB) score while keeping the overall architecture simple and within the prescribed limits.

Key Findings

  • The original starter implementation provided a strong baseline.
  • Weight tying reduced parameter count but consistently increased BPB.
  • Replacing Adam with AdamW did not improve convergence within the fixed 2000-step training budget.
  • The most significant improvement came from replacing the raw byte tokenizer with a byte-level BPE tokenizer.
  • The multilingual corpus (English + Hindi) benefited from BPE tokenization because frequently occurring byte sequences could be represented using fewer tokens.
  • A vocabulary sweep demonstrated that increasing the vocabulary to 786 achieved the best trade-off between sequence compression and prediction complexity.

Conclusion

Among all the evaluated modifications, tokenizer improvements produced the largest performance gains. While optimizer-level changes and weight tying did not improve the model under the fixed training budget, introducing a byte-level BPE tokenizer significantly reduced the BPB. A further vocabulary sweep showed that a vocabulary size of 786 achieved the best overall performance while remaining within the assignment constraints.
